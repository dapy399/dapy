## 🟢 低成本高加分（建议必做）

### 1. 前端工程化规范
**是什么**：ESLint + Prettier + Husky + lint-staged，提交前自动格式化和检查。
**答辩话术**："我在项目中配置了 Git 提交钩子，代码提交前自动跑 ESLint 和 Prettier，保证团队代码风格一致，减少 Code Review 时的格式问题。"
**实现**：10分钟装几个包 + 写配置。

```bash
npm install -D eslint prettier eslint-plugin-react eslint-plugin-react-hooks husky lint-staged
```

### 2. 错误边界（Error Boundary）
**是什么**：React 组件崩溃时不白屏，显示友好错误页面。
**答辩话术**："我在路由层面加了 ErrorBoundary，当某个页面组件报错时，不会导致整个应用崩溃，而是显示友好的错误提示，用户可以点击刷新恢复。"
**实现**：写一个 `ErrorBoundary` 组件，包裹路由即可。

```tsx
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError() { return { hasError: true }; }
  render() {
    if (this.state.hasError) return <Result status="error" title="页面出错了" />;
    return this.props.children;
  }
}
```

### 3. 页面加载骨架屏
**是什么**：页面切换时显示骨架屏，而不是白屏或转圈。
**答辩话术**："路由懒加载时，我用 AntD 的 Skeleton 组件替代了默认的 Spin 加载动画，用户体验更接近真实页面结构，减少等待焦虑感。"
**实现**：把路由 `fallback` 从 `<Spin>` 改成 `<Skeleton active paragraph={{ rows: 10 }} />`。

### 4. 防抖/节流搜索
**是什么**：搜索框输入时不每次按键都发请求，而是等用户停手 300ms 后再搜索。
**答辩话术**："订单列表的搜索框我做了防抖处理，避免用户快速输入时频繁触发请求，减少不必要的网络开销和渲染压力。"
**实现**：用 `lodash-es` 的 `debounce` 包装搜索函数。

```tsx
import { debounce } from 'lodash-es';
const debouncedSearch = useMemo(() => debounce(fetchData, 300), []);
```

### 5. 虚拟滚动表格
**是什么**：表格数据量很大时，只渲染可视区域的行，滚动时动态替换。
**答辩话术**："如果订单数据量达到上万条，普通表格会卡顿。我调研了虚拟滚动方案，后续可以用 `rc-table` 的虚拟滚动或 `react-window` 优化长列表性能。"
**实现**：用 `antd` 的 `virtual` 属性（AntD 5.2 部分表格支持），或引入 `react-window`。

## 🟡 中等成本（选做 2-3 个）

### 6. 国际化（i18n）
**是什么**：支持中英文切换。
**答辩话术**："项目预留了国际化架构，通过 react-i18next 管理翻译文件，菜单、按钮、提示语全部可配置，方便后续出海或多语言团队使用。"
**实现**：装 `react-i18next i18next`，配置语言包，用 `useTranslation()` 替换所有硬编码中文。

### 7. 数据导出 Excel
**是什么**：表格数据一键导出为 .xlsx 文件。
**答辩话术**："订单列表支持导出 Excel，我使用了 sheetjs 库把前端表格数据直接生成 xlsx 文件，不需要后端配合，纯前端完成导出。"
**实现**：装 `xlsx`，写导出函数。

```tsx
import * as XLSX from 'xlsx';
const exportExcel = (data: any[]) => {
  const ws = XLSX.utils.json_to_sheet(data);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, '订单数据');
  XLSX.writeFile(wb, `订单列表_${dayjs().format('YYYYMMDD')}.xlsx`);
};
```

### 8. 水印功能
**是什么**：页面显示当前用户名的半透明水印，防止截图泄密。
**答辩话术**："考虑到后台系统涉及司机和订单敏感数据，我在页面底层加了用户水印，截图可追溯来源，提升数据安全意识。"
**实现**：用 Canvas 画文字水印，重复平铺作为背景图。

### 9. 全键盘快捷键
**是什么**：支持键盘操作，如 `Ctrl+K` 全局搜索、`Esc` 关闭弹窗。
**答辩话术**："我预留了快捷键体系，比如按 Ctrl+K 可以快速唤起搜索，提升高频操作的效率。"
**实现**：监听 `keydown` 事件，匹配组合键。

### 10. 自适应布局
**是什么**：移动端也能看（虽然后台主要用PC，但能响应式是加分项）。
**答辩话术**："布局组件使用了 AntD 的 Grid 栅格系统，搜索表单和统计卡片在窄屏下自动换行，保证在小屏设备上的可用性。"
**实现**：表格、表单都用 `Row/Col` 栅格，表格外层加 `scroll={{ x: true }}`。

### 11. Mock 接口动态拦截
**是什么**：用 MSW（Mock Service Worker）拦截真实 HTTP 请求，而不是直接 import mock 数据。
**答辩话术**："Mock 层我用了 MSW，在浏览器层面拦截 fetch/xhr 请求，返回模拟数据。这样切换到真实后端时，前端代码完全不用改，只需要关闭 MSW 即可。"
**实现**：装 `@mswjs/browser`，配置 handlers。

## 🔴 高成本（如果时间充裕）

### 12. WebSocket 实时通知
**是什么**：右上角消息铃铛有红点，新订单/司机异常实时推送。
**答辩话术**："系统接入了 WebSocket，当有新订单或司机异常时，顶部导航栏的消息铃铛会实时出现红点提示，模拟真实业务场景的实时性。"
**实现**：装 `socket.io-client`，后端配合推送，或者前端用 `setInterval` 模拟。

### 13. 大屏适配（数据可视化驾驶舱）
**是什么**：工作台做成大屏风格，用 rem 或 vw 做自适应缩放。
**答辩话术**："工作台 Dashboard 除了常规展示，还可以适配大屏模式，用 vw/vh 单位做等比缩放，适合放在运营中心的大屏上展示。"
**实现**：用 `postcss-px-to-viewport` 或手写 vw 样式。

### 14. 图片懒加载 + OSS/CDN
**是什么**：用户头像、地图标记图标懒加载，实际项目用 CDN 加速。
**答辩话术**："项目中大量使用了图片资源，实际部署时会接入 CDN 加速，组件层面也用 IntersectionObserver 做了图片懒加载，减少首屏加载时间。"
**实现**：用 `react-lazy-load-image-component` 或原生 `loading="lazy"`。

### 15. 前端监控（Sentry 或自研）
**是什么**：收集前端报错、性能指标、用户行为。
**答辩话术**："实际生产环境中，我会接入 Sentry 或自研监控平台，收集 JS 报错、接口失败率、页面加载时长，第一时间发现问题。"
**实现**：装 `@sentry/react`，配 DSN。
