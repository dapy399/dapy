## 1. 自定义 useRequest Hook（请求层封装）

**技术深度**：⭐⭐⭐⭐⭐

**是什么**：封装一个类似 Ant Design Pro 的 `useRequest` Hook，自动处理 loading、error、防抖、轮询、缓存。

**答辩话术**：
> "我没有在每个页面里重复写 `useState + useEffect + try/catch`，而是封装了一个 `useRequest` Hook，内部自动管理 loading、error、数据缓存，还支持防抖和轮询。这样业务代码只需要关注数据消费，请求逻辑全部收敛到 Hook 里。"

**核心代码**：
```tsx
// utils/useRequest.ts
import { useState, useEffect, useRef, useCallback } from 'react';

interface UseRequestOptions<T> {
  manual?: boolean;           // 是否手动触发
  debounceWait?: number;      // 防抖间隔
  pollingInterval?: number;   // 轮询间隔
  cacheKey?: string;          // 缓存key
}

export function useRequest<T>(
  service: () => Promise<T>,
  options: UseRequestOptions<T> = {}
) {
  const { manual = false, debounceWait, pollingInterval, cacheKey } = options;
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);
  const timerRef = useRef<NodeJS.Timeout>();

  const run = useCallback(async () => {
    setLoading(true);
    setError(null);
    try {
      const result = await service();
      setData(result);
      if (cacheKey) sessionStorage.setItem(cacheKey, JSON.stringify(result));
      return result;
    } catch (e) {
      setError(e as Error);
      throw e;
    } finally {
      setLoading(false);
    }
  }, [service, cacheKey]);

  // 防抖包装
  const debouncedRun = useCallback(() => {
    if (timerRef.current) clearTimeout(timerRef.current);
    timerRef.current = setTimeout(run, debounceWait || 0);
  }, [run, debounceWait]);

  // 轮询
  useEffect(() => {
    if (!pollingInterval) return;
    const interval = setInterval(run, pollingInterval);
    return () => clearInterval(interval);
  }, [run, pollingInterval]);

  // 自动执行
  useEffect(() => {
    if (!manual) run();
  }, [manual, run]);

  return { data, loading, error, run: debounceWait ? debouncedRun : run };
}
```

**用法**：
```tsx
const { data, loading, run } = useRequest(() => fetchOrders(params), {
  debounceWait: 300,
  cacheKey: 'order_list',
});
```

## 2. 路由级权限守卫 + 动态路由

**技术深度**：⭐⭐⭐⭐⭐

**是什么**：不是写死路由表，而是根据后端返回的菜单数据，动态生成路由配置。

**答辩话术**：
> "路由不是前端写死的，而是登录后根据用户角色从后端拉取菜单权限数据，前端动态生成路由表。这样新增菜单或修改权限不需要发版前端，后端配置即可生效。"

**核心逻辑**：
```tsx
// router/index.tsx
import { useEffect, useState } from 'react';

const modules = import.meta.glob('@/views/**/*.tsx');

function generateRoutes(menuList: MenuItem[]) {
  return menuList.map((menu) => ({
    path: menu.path,
    element: lazy(modules[`@/views${menu.component}.tsx`]),
    children: menu.children ? generateRoutes(menu.children) : undefined,
  }));
}
```

**答辩时可以提的深入点**：
- `import.meta.glob` 是 Vite 的特性，编译时收集所有页面组件
- 动态路由 + 懒加载 = 权限控制 + 性能优化同时实现
- 路由守卫做双重校验：菜单权限 + 按钮权限

## 3. 基于 IntersectionObserver 的无限滚动

**技术深度**：⭐⭐⭐⭐

**是什么**：订单列表不是分页按钮翻页，而是滚动到底部自动加载更多。

**答辩话术**：
> "传统分页需要用户点击，打断操作流程。我在订单列表实现了无限滚动，用 IntersectionObserver API 监听列表底部的占位元素，进入视口时自动加载下一页。这比 scroll 事件监听性能更好，不会触发频繁重排。"

**核心代码**：
```tsx
function useInfiniteScroll(callback: () => void) {
  const observerRef = useRef<IntersectionObserver>();
  const targetRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!targetRef.current) return;
    observerRef.current = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) callback();
      },
      { threshold: 0.1 }
    );
    observerRef.current.observe(targetRef.current);
    return () => observerRef.current?.disconnect();
  }, [callback]);

  return targetRef;
}

// 用法
const loadMoreRef = useInfiniteScroll(() => {
  if (!loading && hasMore) fetchNextPage();
});
```

## 4. 虚拟列表（Virtual List）

**技术深度**：⭐⭐⭐⭐⭐

**是什么**：表格数据量极大时，只渲染可视区域的 10-15 行，滚动时动态替换 DOM。

**答辩话术**：
> "当订单数据量达到几万条时，普通表格会直接卡死，因为 DOM 节点太多了。我调研了虚拟滚动方案，核心原理是只渲染视口内的行，通过 `transform: translateY` 模拟滚动位置，DOM 节点数量恒定在十几个，滚动性能从 O(n) 降到 O(1)。"

**深入点可以提**：
- 列表高度不固定时的动态高度计算（getBoundingClientRect）
- 缓冲区的概念（上下各多渲染几条，避免白屏）
- 可以用 `react-window` 或自研，自研更能体现技术深度

## 5. 前端缓存策略（SWR 模式）

**技术深度**：⭐⭐⭐⭐⭐

**是什么**：数据先读缓存展示，同时发请求更新，用户无感知刷新。

**答辩话术**：
> "我实现了一个类似 SWR（stale-while-revalidate）的缓存策略：页面打开时先展示缓存数据，同时静默发请求更新，拿到新数据后替换。用户感知不到加载过程，体验接近原生 App。缓存用 sessionStorage 实现，刷新页面数据不丢失。"

**核心逻辑**：
```tsx
function useSWR<T>(key: string, fetcher: () => Promise<T>) {
  const [data, setData] = useState<T | null>(() => {
    const cached = sessionStorage.getItem(key);
    return cached ? JSON.parse(cached) : null;
  });

  useEffect(() => {
    fetcher().then((newData) => {
      setData(newData);
      sessionStorage.setItem(key, JSON.stringify(newData));
    });
  }, [key, fetcher]);

  return { data, isLoading: data === null };
}
```

## 6. 自定义指令式组件（命令式调用 Modal）

**技术深度**：⭐⭐⭐⭐

**是什么**：不用 JSX 写 Modal，而是像 `Modal.confirm()` 那样命令式调用。

**答辩话术**：
> "AntD 的 Modal 支持命令式调用，但我需要自定义弹窗风格。我实现了一个命令式弹窗管理器，通过 `ReactDOM.createPortal` 动态挂载到 body，支持 `open()`、`close()`、`update()` 方法。业务代码里一行代码就能唤起弹窗，不用在组件里维护 visible 状态。"

**核心代码**：
```tsx
// components/ModalManager.tsx
class ModalManager {
  private container: HTMLDivElement;
  private root: any;

  constructor() {
    this.container = document.createElement('div');
    document.body.appendChild(this.container);
    this.root = createRoot(this.container);
  }

  open(config: ModalConfig) {
    this.root.render(
      <Modal open={true} onCancel={config.onCancel} {...config}>
        {config.content}
      </Modal>
    );
  }

  close() {
    this.root.render(null);
  }
}

export const modal = new ModalManager();

// 用法
modal.open({
  title: '确认删除',
  content: '确定要删除这条订单吗？',
  onOk: () => handleDelete(),
});
```

## 7. 性能优化：React.memo + useMemo + useCallback 组合拳

**技术深度**：⭐⭐⭐⭐

**是什么**：表格行、图表、复杂组件做精细化渲染控制。

**答辩话术**：
> "订单列表每行有 8 个单元格 + 4 个操作按钮，如果父组件状态变化导致 50 行全部重渲染，性能开销很大。我对表格行组件用 `React.memo` 包裹，配合 `useMemo` 缓存列配置、`useCallback` 缓存事件处理函数。只有当前行数据变化时才重渲染，其他行跳过 Diff，列表滚动明显更流畅。"

**可以现场演示**：
- 打开 React DevTools Profiler
- 对比优化前后的渲染次数
- 用 `console.count('render')` 直观展示

## 8. Web Worker 处理大数据

**技术深度**：⭐⭐⭐⭐⭐

**是什么**：大数据计算（如导出 10 万条订单）放到 Web Worker 线程，不阻塞 UI。

**答辩话术**：
> "导出 Excel 时如果数据量很大，前端处理会阻塞主线程，页面直接卡死。我用 Web Worker 把数据转 Excel 的逻辑放到子线程执行，主线程完全不受影响，用户可以继续操作页面，处理完成后通过 postMessage 传回主线程下载。"

**核心代码**：
```ts
// utils/exportWorker.ts
self.onmessage = (e) => {
  const { data, filename } = e.data;
  const ws = XLSX.utils.json_to_sheet(data);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, 'Sheet1');
  const blob = XLSX.write(wb, { bookType: 'xlsx', type: 'array' });
  self.postMessage({ blob, filename });
};

// 使用
const worker = new Worker(new URL('./exportWorker.ts', import.meta.url));
worker.postMessage({ data: orders, filename: 'orders.xlsx' });
worker.onmessage = (e) => {
  downloadBlob(e.data.blob, e.data.filename);
};
```

## 12. 前端路由守卫的深入设计

**技术深度**：⭐⭐⭐⭐

**是什么**：不仅是判断是否登录，还包括权限动态校验、路由白名单、403/404 状态码处理。

**答辩话术**：
> "路由守卫不是简单判断 token 存在就放行。我设计了三层校验：第一层校验 token 有效性和过期时间；第二层校验当前路由是否在用户权限菜单中；第三层校验按钮级权限。如果用户通过 URL 直接访问无权限页面，不是跳首页，而是显示 403 无权访问页面，体验更专业。"

```tsx
// 三层守卫
1. TokenGuard      // token 有效性
2. MenuGuard       // 菜单权限
3. ButtonGuard     // 按钮权限（渲染层面）
```
